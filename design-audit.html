Design System Audit · April 2026

Job Agent Frontend Review

Evaluated against Laws of UX, Material Design 3, Apple HIG, and the design-system.md spec (premium/minimalist, dark-mode-first, gradient accents).

Overall Verdict
Functional and well-structured, but visually generic. The UI relies on Tailwind defaults + system fonts + emoji icons, producing a "good internal tool" aesthetic — not the premium, Apple/CRED-inspired direction your design-system.md targets. The bones are solid (responsive grids, semantic color system, component reuse). The skin needs work (typography, motion, depth, iconography, dark mode).

1 Color System

Brand Palette PASS

Brand indigo (#6366F1) is well-defined with proper tonal scale (50–700). Three interaction states: rest, hover, active. Consistent usage across primary buttons, active nav tabs, and focus rings.

Semantic Colors PASS

Clean mapping: emerald = success/high score, amber = warning/medium, red = danger/low, blue = info. Each has bg/border/text variants. The scoreColor(), sectorColor(), and stageColor() utility functions enforce this consistently.

No Gradient Accents GAP

Your design-system.md spec calls for "gradient accents, not flat colors." The current UI is entirely flat — solid background fills, no gradients anywhere. Score badges, buttons, and header are all single-color.

No Dark Mode FAIL

Design-system.md specifies "dark mode first." The app has no dark mode, no prefers-color-scheme media query, no theme toggle. All backgrounds are hardcoded white/gray.

2 Typography

No Custom Fonts FAIL

Uses Tailwind's default system font stack (system-ui, sans-serif). No display font, no body font pairing. The design-system.md spec requires a defined type scale: Display, Headline, Title, Body, Label — with explicit sizes, weights, and line heights. None of this exists.

Type Scale INCOMPLETE

Ad-hoc Tailwind size classes: text-xs, text-sm, text-base, text-lg. No defined hierarchy. Most UI text is text-xs/text-sm, making the interface feel uniformly small. Headers like "Open Roles" and page titles don't stand out enough from body content.

3 Spacing & Layout

4px Grid PASS

Tailwind's spacing scale is 4px-based by default. Usage of px-3 (12px), py-2 (8px), gap-4 (16px) etc. stays on the grid. Consistent throughout.

Responsive Grid PASS

Proper responsive breakpoints: 1 col mobile → 2 col tablet (sm:640px) → 3 col desktop (lg:1024px) → 4 col wide (xl:1280px). Applied correctly across OpenRoles grid, Sources grid, HowItWorks cards.

Container Width PASS

max-w-7xl mx-auto (1280px) is appropriate. Content doesn't stretch uncomfortably on wide screens.

Elevation System MINIMAL

Only 2 levels used: shadow-sm (cards at rest) and shadow-md (cards on hover). MD3 recommends 5 elevation levels. No elevation hierarchy distinguishes sticky header, modals, or dropdowns from content cards.

4 Components

Button System PASS

Six button variants defined in index.css: primary, secondary, ghost, success, warning, danger. All share a base .btn class. Proper disabled states with opacity + cursor. Size is consistent (text-xs, px-3 py-1.5).

  • Missing: icon-only button variant, FAB, loading state with spinner
  • Missing: size variants (sm, md, lg) — currently one size fits all

Card Component PASS

Single .card class: white bg, rounded-xl, gray-200 border, shadow-sm. Used consistently for JobCard and various page cards. Hover state adds shadow-md — good depth signal.

  • Missing: card variants (elevated, filled, outlined per MD3)
  • Missing: card skeleton/loading state

Iconography FAIL

Entire app uses emoji for icons: ⚡ (logo), 🔍📊🗂✅ (how-it-works), ✉️ (outreach), ⏰ (follow-up). Emojis render inconsistently across platforms and don't scale cleanly. They undermine the premium aesthetic target.

Recommendation: Replace with a proper icon library (Lucide, Phosphor, or SF Symbols-inspired set). 24px stroke icons, consistent weight.

Form Inputs BASIC

Text inputs and textareas have focus rings (focus:ring-2 focus:ring-brand-200) and basic styling. No outlined vs. filled variants, no floating labels, no validation state styling (error borders, helper text). The skip-reason input and notes fields lack visual hierarchy.

Navigation FUNCTIONAL

Sticky header with 6 horizontal tabs. Active tab uses brand-500 underline. No mobile hamburger menu — tabs overflow on small screens. No breadcrumbs or secondary navigation. The "HowItWorks" collapsible is an unusual pattern for a nav-adjacent element.

5 UX Principles (Laws of UX)

Fitts's Law PARTIAL

Buttons use px-3 py-1.5 — approximately 36px height, under the 44px minimum touch target. Action buttons in JobCard footer are small and closely spaced. The "Skip" text link is especially small and hard to tap on mobile.

Hick's Law PASS

Good progressive disclosure. Each page focuses on one status category. Filters limit visible options. Pipeline and OnRadar use collapsible cards to hide detail until needed. The Sources page groups actions into labeled sections.

Aesthetic-Usability Effect UNDER-LEVERAGED

The interface is clean but not beautiful. System fonts + emoji icons + flat white surfaces create a functional-but-forgettable experience. Users perceive beautiful interfaces as more usable — investing in typography, depth, and subtle motion would improve perceived quality.

Doherty Threshold NO FEEDBACK

No loading indicators, no optimistic UI, no skeleton screens. Status changes (Pipeline, Reached Out, Applied) happen silently — no toast, no animation, no confirmation. The "Loading..." text string is the only loading state. Users have no confidence their action registered.

6 Accessibility (WCAG 2.1 AA)

Color Contrast PASS

Primary text (gray-900 on white) exceeds 4.5:1. Brand-500 on white is 4.6:1 — borderline but passes. Score badges pair color with text labels, not color-only.

Touch Targets FAIL

Buttons at ~36px height are below the 44x44px minimum. Text links ("Skip", "View") are even smaller. Filter dropdowns have adequate size but tag pills in cards are tight. Needs padding increase on all interactive elements.

ARIA & Semantics MINIMAL

No ARIA labels, roles, or live regions. Tables use <thead>/<tbody> (good). No <nav> landmark for navigation. No aria-expanded on collapsible sections. Screen reader experience would be poor.

Focus Management BASIC

Focus rings on inputs (focus:ring-2 focus:ring-brand-200) but focus:outline-none removes the browser default without always replacing it on buttons and links. Tab order follows DOM order (acceptable). No focus trap in skip-reason form or outreach draft areas.

7 Motion & Transitions

Hover Transitions MINIMAL

Only transition-colors on buttons and transition-all on card shadows. No entrance animations, no page transitions, no staggered reveals. Collapsible sections (OnRadar, Pipeline) toggle instantly with no height animation. Status changes produce no visual feedback.

The design-system.md spec defines 4 duration levels (100ms, 200ms, 300ms, 500ms) with easing curves — none are implemented.

8 Recommendations
1. Add a type system

Pick a display + body font pairing. Define a 5-level type scale in tailwind.config.js. Apply consistently: page titles = Headline, card titles = Title, metadata = Label.

Effort: ~2 hours · Impact: High
2. Replace emoji with icons

Install Lucide React or Phosphor. Replace all emoji usage with 20px/24px stroke icons. Consistent visual language, better cross-platform rendering.

Effort: ~3 hours · Impact: High
3. Add dark mode

Convert all color values to CSS variables. Add dark: variants via Tailwind's dark mode. Add a theme toggle in the header. Your spec says dark-mode-first.

Effort: ~4 hours · Impact: High
4. Action feedback

Add toast notifications for status changes. Loading spinners on async buttons. Skeleton screens instead of "Loading..." text. Optimistic UI updates.

Effort: ~3 hours · Impact: Medium
5. Touch target sizing

Increase button padding to min 44px height. Add hit-area padding to text links. Ensure filter dropdowns and tag pills meet minimum targets.

Effort: ~1 hour · Impact: Medium
6. Entrance motion

Add staggered fade-in on card grids (animation-delay per card). Smooth height transitions on collapsibles. Page transition fade. Use the 4 duration tokens from your spec.

Effort: ~2 hours · Impact: Medium
7. Elevation hierarchy

Define 5 shadow levels. Sticky header at level 2, cards at level 1, modals at level 4. Add gradient accents per your spec — subtle brand gradient on header or primary buttons.

Effort: ~1 hour · Impact: Low-Medium
8. ARIA & semantics

Add <nav> landmark, aria-expanded on collapsibles, aria-live on toast regions, proper button labels for icon-only elements.

Effort: ~2 hours · Impact: Low (until users need it)
